# 04 — Lenses

A **lens** adapts how Brand Truth is *applied* for a specific audience or context — B2B vs. consumer, web vs. print, onboarding vs. support — **without changing the truth itself**.

## The rule that matters
Lenses **emphasize, sequence, and adjust**; they never contradict. If a lens needs to break a truth, that's not a lens — it's a sign the truth is wrong, or you need a new audience entirely. The imprint, voice, and design truths remain the single source; a lens is a configured *view* onto them.

## What goes in a lens
- The audience/context it serves and what they care about
- Which truths to **emphasize** and which to soft-pedal
- Tone and channel adjustments within the allowed range
- Concrete do/don't examples for that context

## How to author
- One lens per file (e.g. `example-consumer.md`, `b2b.md`, `print.md`).
- Carry the truth frontmatter contract; include a `**Why:**`.
- Reference the truths you're flexing rather than restating them.
